A S3BucketDol that completely avoids the bucket exists check. This is needed for Supabase endpoints where the bucket is not created until the first object is uploaded, for example.

Parameters:

  • bucket_name – name of bucket to store data in

  • make_bucket – if True, create bucket if it does not exist. If None, skip bucket existence checks completely. If False, check for existence but don’t create.

  • path – prefix to use for bucket keys

  • aws_access_key_id – AWS access key ID

  • aws_secret_access_key – AWS secret access key

  • aws_session_token – AWS session token

  • endpoint_url – URL of S3 endpoint

  • region_name – AWS region name

  • profile_name – AWS profile name

Validate bucket name and create if needed

If make_bucket is None, skip existence check entirely. If make_bucket is True, create bucket if it doesn’t exist. If make_bucket is False, check existence but don’t create.
